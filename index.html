Overview

The .Trends API offers a robust array of data related to website traffic and market trends. With access to the .Trends API, you can easily pull raw Semrush .Trends data straight into your in-house reports or systems.

You can integrate the .Trends API with:

  • Dashboards: Enterprise portals, CRMs, ERPs
  • Reporting software: Tableau, Power BI, and more
  • Spreadsheets: Google Sheets and Microsoft Excel
  • Other custom in-house analytics systems

Key benefits

  • Consolidate all your data sources into one location for easier analysis and quicker market and competitive insights.
  • Access your data from a variety of platforms for convenient visualization and streamlined decision-making.
  • Automate and customize your data reporting to quickly reveal the unique insights your business needs to take the lead.
  • Make detailed, customizable data queries to extract precisely the information you need, enhancing your analysis capabilities.
  • Accelerate the development and deployment of data-driven applications by integrating robust sets of API data directly into your software.

Enables the Site Audit Tool for a project.

This request lets you enable the Site Audit Tool for a project to schedule audits, include or exclude pages from the crawl, and set the number of pages to crawl. About the Site Audit Configuration

path Parameters
ID
required
integer

Project ID. Learn how to get your project ID.

query Parameters
key
required
string

The API key for your Semrush account.

Request Body schema: application/json
required
domain
string

The project URL.

scheduleDay
integer

Specifies the day of the week (from 1 to 7) for periodic execution; use 8 for manual start.

notify
boolean

Specifies whether an email notification is sent after the audit is completed.

allow
Array of strings

Masks the Allow directive in the project.

disallow
Array of strings

Masks the Disallow directive in the project.

pageLimit
integer

The number of crawled pages.

userAgentType
integer

The type of user agent. Available values: 8: SiteAuditBot Mobile 7: SiteAuditBot Desktop 3: GoogleBot Mobile 2: GoogleBot Desktop 9: OpenAl-Search

removedParameters
Array of strings

Specifies URL parameters to be excluded from the audit scope.

crawlSubdomains
boolean

Specifies whether to crawl subdomains of the selected domain. Available values: True: SemrushBot will crawl the selected domain and its subdomains. False: SemrushBot will crawl only the selected domain.

respectCrawlDelay
boolean

Specifies whether SemrushBot will follow the Crawl-delay directive in robots.txt. Available values: True: SemrushBot will follow the Crawl-delay directive in robots.txt. False: SemrushBot will crawl pages with a one-second interval.

Responses

https://api.semrush.com/management/v1/projects/{ID}/siteaudit/enable

Domain Overview (All Databases)

Returns keyword rankings and metrics for the specified domain from all available regional databases. Costs 10 API units per line of info. About the Domain Overview report ›

query Parameters
type
required
string
Value: "domain_ranks"
Example: type=domain_ranks

Report type. Must be domain_ranks.

key
required
string
Example: key=API_KEY

API key from your Profile page.

domain
required
string
Example: domain=apple.com

Domain name to investigate.

database
string
Example: database=us

Regional database (e.g., us, uk). Databases ›

display_limit
integer <= 4000000
Default: 10000
Example: display_limit=10000

Number of results to return.

display_offset
integer
Default: 0
Example: display_offset=0

Skip this number of lines before starting output.

export_escape
integer
Value: 1
Example: export_escape=1

If set to 1, wraps output columns in double quotes.

display_date
string^\d{6}15$
Example: display_date=20240315

Month to show report for. Format: YYYYMM15. Example: 20240315 for March 2024.

export_columns
string
Example: export_columns=Db,Dn,Rk,Or,Ot,Oc,Ad,At,Ac,Sh,Sv

Comma-separated list of columns to export. Columns ›

string
Enum: "rk_asc" "rk_desc" "or_asc" "or_desc" "ot_asc" "ot_desc"
